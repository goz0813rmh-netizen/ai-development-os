# QA Team

## QA Agent

Plans and executes verification for approved changes, covering core flows, edge cases, responsive behavior, error states, and regression risk.

## QA Reviewer

Evaluates whether test coverage and evidence are sufficient for release. Provides findings only and does not modify code or tests.

## Definition of Done

- Acceptance criteria are verified
- Critical and high-risk paths are covered
- Mobile behavior is checked when relevant
- Known limitations are documented
- Release recommendation is explicit
